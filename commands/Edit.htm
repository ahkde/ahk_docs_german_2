Öffnet das aktuelle Skript mit dem zugeordneten Editor.

Edit

Die Edit-Funktion öffnet das aktuelle Skript mithilfe des zugeordneten Verbs "edit", das in der Registry angegeben ist (oder mit dem Windows-Texteditor, wenn kein Verb vorhanden ist). Stellt sich heraus, dass ein Editorfenster das Skript bereits offen hat (abhängig vom Fenstertitel), wird dieses Fenster aktiviert, anstatt eine neue Instanz des Editors zu öffnen.

Diese Funktion hat keine Wirkung, wenn sie innerhalb eines kompilierten Skripts aufgerufen wird.

Verschiedene Texteditoren unterstützen eine Syntaxhervorhebung für AutoHotkey (siehe unten). Des Weiteren kann eine kontextabhängige Hilfe für AutoHotkey-Funktionen mit diesem Beispiel aktiviert werden. Um die eigene Produktivität zu erhöhen, kann ein Auto-Vervollständigungs-Tool wie das Skript von boiler oder das Skript von Helgef genutzt werden, die bei fast jedem Editor funktionieren. Es überwacht, was Sie eingeben, und zeigt Menüs und Parameterlisten an, die Ihnen einen Teil der Eingabe abnimmt und Sie an die Reihenfolge der Parameter erinnert.

Beispiele

Öffnet das Skript zum Editieren.

Edit

Wenn die Befehlszeilenverwendung Ihres Editors ungefähr wie Editor.exe "Kompletter Pfad des Skripts.ahk" aussieht, können Sie ihn wie folgt als Standardeditor für AHK-Dateien festlegen. Wenn Sie das Skript ausführen, werden Sie aufgefordert, die EXE-Datei Ihres Editors auszuwählen.

Editor := FileSelect(2,, "Wählen Sie Ihren Editor aus", "Programme (*.exe)")
if Editor = ""
    ExitApp
RegWrite "REG_SZ", "HKCR\AutoHotkeyScript\Shell\Edit\Command",, Format('"{1}" "%1"', Editor)

Editoren mit AutoHotkey-Unterstützung

SciTE4AutoHotkey ist eine benutzerdefinierte Version des Texteditors SciTE, der speziell zum Editieren von AutoHotkey-Skripten entwickelt wurde. Seine Features umfassen:

  • Syntax-Hervorhebung
  • Intelligente automatische Einrückung
  • Automatische Vervollständigung
  • Aufruftipps (auch als IntelliSense bekannt)
  • Code-Folding
  • Unterstützung für interaktive Debuggen
  • Weitere Tools für AutoHotkey-Skripte

SciTE4AutoHotkey kann hier gefunden werden: http://fincs.ahk4.net/scite4ahk/

AHK Studio ist ein Editor für AutoHotkey-Skripte, der mit AutoHotkey erstellt wurde. Im folgenden Forum-Thread finden Sie genaue Details, Video-Anleitungen und eine ständig wachsende Liste von Features: AHK Studio

AutoGUI ist eine integrierte Entwicklungsumgebung für AutoHotkey, die einen GUI-Designer mit einem Skripteditor verknüpft. Sie finden es hier: AutoGUI - GUI Designer and Script Editor

Weitere Editoren, für die die Syntaxhervorhebung für AutoHotkey aktiviert werden kann:

  • AkelPad
  • Crimson Editor
  • Eclipse, FAR-Manager, und andere Editoren, die Colorer take5 verwenden
  • Emacs
  • Notepad++
  • Notepad2
  • PSPad
  • SciTE und möglicherweise andere Scintilla-basierte Editoren
  • Sublime Text Editor
  • Total Commander mit Synplus-Plugin
  • Visual Studio Code mit dem AutoHotkey-Extension von cweijan

Zusätzlich enthält die Zip-Version von AutoHotkey Basic (https://www.autohotkey.com/download/1.0/) Dateien zum Aktivieren der Syntaxhervorhebung für folgende Editoren. Allerdings sind einige dieser Dateien veraltet und funktionieren möglicherweise nicht mehr:

  • ConTEXT
  • EditPlus
  • EmEditor
  • jEdit
  • MED
  • TextPad
  • UltraEdit
  • Vim
