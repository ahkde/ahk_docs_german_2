Loop

Führt eine Reihe von Funktionen wiederholt aus - entweder eine bestimmte Anzahl von Wiederholungen oder bis ein Break auftritt.

Loop Anzahl

Parameter

Anzahl

Typ: Integer

Wie viele Male (Iterationen) die Schleife durchgeführt werden soll. Lässt man diesen Parameter weg, wird die Schleife endlos fortgesetzt, bis ein Break oder Return auftritt. Beachten Sie aber, dass ein explizit leerer Wert oder eine Zahl kleiner als 1 die Schleife komplett überspringen lässt.

Anzahl wird nur einmal ausgewertet, bevor die Schleife beginnt. Wenn Anzahl beispielsweise ein Ausdruck mit Nebeneffekten wie Funktionsaufrufe oder Zuweisungen ist, dann treten diese Nebeneffekte nur einmal auf.

Wenn Anzahl mit runden Klammern umschlossen wird, ist ein Leer- oder Tabulatorzeichen nicht notwendig. Zum Beispiel: Loop(2)

Bemerkungen

Nach einer Schleife erfolgt üblicherweise ein Block (eine Sammlung von Anweisungen), der den Schleifenkörper definiert. Eine Schleife mit nur einer einzigen Anweisung benötigt keinen Block (ein "if" und das dazugehörige "else" werden hierbei als Einzelanweisung angesehen).

Diese Anweisung wird häufig als Endlosschleife benutzt; mit der Break-Anweisung irgendwo im Schleifenkörper kann man bestimmen, wann die Schleife gestoppt werden soll.

Break und Continue sind besser geeignet als Goto, weil sie in der Regel die Übersichtlichkeit und Wartbarkeit eines Skripts erhöhen. Man kann auch eine "While" oder "Do...While/Until"-Schleife simulieren, indem man die erste oder letzte Anweisung im Schleifenkörper zu einer IF-Anweisung macht, die in Abhängigkeit einer Bedingung die Break-Anweisung ausführt. In der Regel sollte man aber While oder Loop...Until benutzen.

Die interne Variable A_Index enthält die Nummer des aktuellen Schleifendurchlaufs. Sie enthält eine 1, wenn der Schleifenkörper das erste Mal durchgeführt wurde. Beim zweiten Mal enthält sie eine 2; und so weiter. Wenn eine innere Schleife von einer äußeren Schleife umschlossen ist, hat die innere Schleife Vorrang. A_Index funktioniert in jeder Schleifenvariante, einschließlich Datei-Schleifen und Registry-Schleifen; außerhalb einer Schleife enthält A_Index eine 0.

Man kann in A_Index einen beliebigen Integer-Wert speichern. Wenn Anzahl angegeben ist, beeinflusst eine Änderung von A_Index die aktuelle Anzahl der Durchläufe, die durchgeführt werden. Zum Beispiel würde A_Index := 3 eine Schleife dazu bringen, sich so zu verhalten, als wäre sie im dritten Durchlauf (A_Index wird im nächsten Durchlauf eine 4 sein), während A_Index-- verhindert, dass die Schleife jemals den nächsten Durchlauf erreicht.

Nach der Schleife kann optional eine Else-Anweisung erfolgen, die ausgeführt wird, wenn Anzahl Null ist.

Der One True Brace Style (OTB) kann je nach Bedarf angewendet werden. Zum Beispiel:

Loop {
    ...
}
Loop WiederholungenAnzahl {
    ...
}

Spezialisierte Schleifen: Mit Schleifen können Dateien, Ordner oder Registry-Einträge automatisch abgerufen werden (einzeln). Siehe Datei-Schleife und Registry-Schleife für Details. Dateilesende Schleifen können den kompletten Inhalt einer Datei zeilenweise durchgehen. Parsende Schleifen können einzelne Felder einer Zeichenkette durchgehen.

Beispiele

Erstellt eine Schleife mit 3 Iterationen (Wiederholungen).

Loop 3
{
    MsgBox "Iterationsnummer ist " A_Index  ; A_Index wird 1, 2 und 3 sein.
    Sleep 100
}

Erstellt eine Endlosschleife, die aber nach der 25. Iteration unterbrochen wird.

Loop
{
    if (A_Index > 25)
        break  ; Unterbricht die Schleife
    if (A_Index < 20)
        continue ; Überspringt das darunter und startet eine neue Iteration
    MsgBox "A_Index = " A_Index ; Dies wird nur die Nummern von 20 bis 25 anzeigen
}
