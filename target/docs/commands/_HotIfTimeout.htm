#HotIfTimeout

Bestimmt die maximale Zeit, die zur Auswertung eines einzelnen #HotIf-Ausdrucks aufgebracht werden soll.

#HotIfTimeout Zeitlimit

Parameter

Zeitlimit

Typ: Integer

Zeitlimit-Wert in Millisekunden, der global gelten soll.

Bemerkungen

Ein Zeitlimit wurde implementiert, um lang laufende Ausdrücke daran zu hindern, den Tastatureingabeprozess hinauszuzögern. Sobald das Zeitlimit überschritten ist, wird der Ausdruck weiter ausgewertet, aber der Tastatur-Hook fortgesetzt, als hätte der Ausdruck bereits False zurückgegeben.

Wenn diese Direktive im Skript nicht angegeben ist, wird sie sich so verhalten, als wäre sie auf 1000 gesetzt.

Beachten Sie, dass das System sein eigenes Zeitlimit implementiert, das im DWORD-Wert LowLevelHooksTimeout des folgenden Registry-Schlüssels definiert ist:

HKEY_CURRENT_USER\Control Panel\Desktop

Wenn das Zeitlimit des Systems überschritten ist, kann es passieren, dass das System den Tastatur-Hook des Skripts nicht mehr aufrufen wird und dass Hook-Hotkeys dementsprechend nicht mehr funktionieren werden, bis der Hook neu registriert oder das Skript neu geladen wurde. Der Hook kann üblicherweise neu registriert werden, indem man alle Hotkeys via Suspend deaktiviert und wieder aktiviert.

Genaueres zu diesem Zeitlimit ist in der Dokumentation von Microsoft nicht zu finden, aber Untersuchungen haben ergeben, dass für Windows 7 und höher Folgendes gilt: Wenn LowLevelHooksTimeout nicht definiert ist, gilt standardmäßig ein Zeitlimit von 300 ms. Der Hook kann maximal 10-mal das Zeitlimit überschreiten, bevor er stillschweigend entfernt wird.

Wenn ein bestimmter Hotkey mehrere #HotIf-Varianten hat, bekommt jede Variante ein eigenes Zeitlimit zugewiesen, wodurch sich die Chance erhöht, dass das Zeitlimit des Systems überschritten wird. Dieser Sachverhalt wird eventuell in einer zukünftigen Version geändert.

#HotIfTimeout kann wie alle anderen Direktiven nicht bedingt ausgeführt werden.

Beispiele

Setzt das #HotIf-Zeitlimit auf 10 ms anstelle von 1000 ms.

#HotIfTimeout 10
