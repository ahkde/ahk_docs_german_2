HasMethod

Gibt eine Nummer ungleich 0 zurück, wenn ein bestimmter Wert eine Methode mit einem bestimmten Namen hat.

HatMethode := HasMethod(Wert , Name, ParamAnzahl)

Parameter

Wert

Typ: Any

Ein Wert beliebigen Typs, außer vom Typ ComObject.

Name

Typ: Zeichenkette

Der zu prüfende Methodenname.

Lassen Sie diesen Parameter weg, um zu prüfen, ob Wert selbst aufrufbar ist.

ParamAnzahl
Typ: Integer

Die Anzahl der Parameter, die voraussichtlich an die Methode oder Funktion übergeben werden. Wenn dieser Parameter angegeben ist, können die Eigenschaften MinParams, MaxParams und IsVariadic der Methode abgefragt werden, um zu verifizieren, ob sie diese Anzahl von Parametern akzeptieren kann. Wenn diese Eigenschaften nicht vorhanden sind, wird die Anzahl der Parameter nicht verifiziert.

Diese Anzahl sollte den impliziten this-Parameter nicht enthalten.

Lässt man diesen Parameter weg (oder wurde die Anzahl der Parameter nicht verifiziert), wird standardmäßig überprüft, ob eine Call-Methode vorhanden ist, um zu verifizieren, ob das Objekt sehr wahrscheinlich aufrufbar ist.

Rückgabewert

Typ: Integer (boolescher Wert)

Diese Funktion gibt True zurück, wenn eine Methode gefunden wurde und die Validierung bestanden hat (falls durchgeführt), ansonsten False.

Bemerkungen

HasMethod hat die gleichen Einschränkungen wie GetMethod.

Diese Funktion kann genutzt werden, um abzuschätzen, ob ein Wert eine bestimmte Aktion unterstützt. Werte ohne Call-Methode beispielsweise können nicht aufgerufen oder an SetTimer übergeben werden, während Werte ohne __Enum-Methode oder Call-Methode nicht an For übergeben werden können. Die Existenz einer Methode garantiert jedoch nicht, dass sie aufgerufen werden kann, da es Anforderungen gibt, die erfüllt sein müssen, wie z. B. Parameteranzahl.

Wenn ParamAnzahl angegeben ist, entspricht die Validierung, die diese Funktion durchführt, der Validierung, die von internen Funktionen wie SetTimer durchgeführt wird.

Ein Rückgabewert von False bedeutet nicht unbedingt, dass die Methode nicht aufgerufen werden kann, da der Wert eine __Call-Metafunktion haben kann. Allerdings wird __Call in bestimmten Zusammenhängen nicht ausgelöst, z. B. wenn __Enum von For aufgerufen wird. Wenn __Call vorhanden ist, gibt es keine Möglichkeit, zu erkennen, welche Methoden es eventuell unterstützt.

Diese Funktion unterstützt primitive Werte.

Beispiele

Veranschaulicht die Verwendung dieser Funktion.

MsgBox HasMethod(0, "HasMethod") ; 1
MsgBox HasMethod(0, "Call") ; 0
