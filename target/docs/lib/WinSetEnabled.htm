WinSetEnabled

Aktiviert oder deaktiviert die Interaktion mit einem bestimmten Fenster.

WinSetEnabled Wert , FensterTitel, FensterText, AusnahmeTitel, AusnahmeText

Parameter

Wert

Typ: Integer

Geben Sie einen der folgenden Werte an:

  • 1 oder True schaltet die Einstellung ein
  • 0 oder False schaltet die Einstellung aus
  • -1 setzt die Einstellung auf den entgegengesetzten Zustand
FensterTitel, FensterText, AusnahmeTitel, AusnahmeText

Typ: Zeichenkette, Integer oder Objekt

Wenn jeder dieser Parameter ungesetzt ist oder weggelassen wird, wird das Zuletzt Gefundene Fenster verwendet. Andernfalls geben Sie für FensterTitel ein Fenstertitel oder andere Kriterien zur Identifizierung des Zielfensters und/oder für FensterText eine Teilzeichenkette aus einem Textelement des Zielfensters an (Textelemente können mit dem internen Tool "Window Spy" aufgedeckt werden). Versteckte Textelemente werden erkannt, wenn DetectHiddenText eingeschaltet wurde. Fenster, deren Titel oder Texte AusnahmeTitel bzw. AusnahmeText enthalten, werden ignoriert.

Fehlerbehandlung

Es wird ein TargetError ausgelöst, wenn das Fenster nicht gefunden werden konnte.

Es wird ein OSError ausgelöst, wenn die Änderung nicht angewendet werden konnte.

Bemerkungen

Wenn die Interaktion mit einem Fenster verboten ist, kann der Benutzer es weder verschieben noch mit dessen Steuerelementen interagieren. Außerdem werden solche Fenster in der Alt-Tab-Liste nicht angezeigt.

Beispiel #1 auf der WinGetStyle-Seite kann verwendet werden, um zu prüfen, ob die Interaktion mit einem Fenster aktuell deaktiviert ist.

Die ID des Fensters, das sich unter dem Mauszeiger befindet, kann mit MouseGetPos abgerufen werden.

Fenstertitel und -texte sind Groß-/Kleinschreibung-sensitiv. Versteckte Fenster werden nur erkannt, wenn DetectHiddenWindows eingeschaltet ist.
