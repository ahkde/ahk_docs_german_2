Map-Objekt

class Map extends Object

Ein Map-Objekt ordnet eine Reihe von Werten, genannt Schlüssel, einer anderen Gruppe von Werten zu. Ein Schlüssel mit seinem zugeordneten Wert wird als Schlüssel-Wert-Paar bezeichnet. Ein Map-Objekt kann beliebig viele Schlüssel-Wert-Paare enthalten, allerdings muss jeder Schlüssel einzigartig sein.

Ein Schlüssel kann ein Integer, eine Objektreferenz oder eine null-terminierte Zeichenkette sein. Der Vergleich von Zeichenketten-Schlüsseln ist Groß-/Kleinschreibung-sensitiv, während Objekte via Referenz/Adresse verglichen werden. Float-Schlüssel werden automatisch in eine Zeichenkette umgewandelt.

Der einfachste Weg, ein Map-Objekt zu nutzen, besteht darin, ein Schlüssel-Wert-Paar über die implizite __Item-Eigenschaft abzurufen oder zu setzen, indem man einfach neben dem Map-Objekt den Schlüssel in eckigen Klammern setzt. Zum Beispiel:

Farben := Map()
Farben["Rot"] := "ff0000"
Farben["Grün"] := "00ff00"
Farben["Blau"] := "0000ff"
for Farbe in Array("Blau", "Grün")
    MsgBox Farben[Farbe]

"MapObj" wird hier stellvertretend für ein beliebiges Map-Objekt verwendet, da "Map" die Klasse selbst ist.

Zusätzlich zu den Methoden und Eigenschaften, die von Object geerbt werden, haben Map-Objekte die folgenden vordefinierten Methoden und Eigenschaften.

Inhaltsverzeichnis

  • Statische Methoden:
    • Call: Erstellt ein Map-Objekt und setzt Elemente.
  • Methoden:
    • Clear: Entfernt alle Schlüssel-Wert-Paare aus einem Map-Objekt.
    • Clone: Gibt eine flache Kopie eines Map-Objekts zurück.
    • Delete: Entfernt ein Schlüssel-Wert-Paar aus einem Map-Objekt.
    • Get: Gibt den Wert eines zugehörigen Schlüssels oder einen Standardwert zurück.
    • Has: Gibt True zurück, wenn der angegebene Schlüssel einen zugeordneten Wert innerhalb des Map-Objekts hat.
    • Set: Setzt null oder mehr Elemente.
    • __Enum: Enumeriert Schlüssel-Wert-Paare.
    • __New: Setzt Elemente. Äquivalent zu Set.
  • Eigenschaften:
    • Count: Ruft die Anzahl von Schlüssel-Wert-Paaren in einem Map-Objekt ab.
    • Capacity: Ermittelt oder setzt die aktuelle Kapazität eines Map-Objekts.
    • CaseSense: Ermittelt oder setzt die Groß-/Kleinschreibung-Sensitivität eines Map-Objekts.
    • Default: Definiert den Standardwert, der zurückgegeben werden soll, wenn ein Schlüssel nicht gefunden wird.
    • __Item: Ermittelt oder setzt den Wert eines Schlüssel-Wert-Paares.

Statische Methoden

Call

Erstellt ein Map-Objekt und setzt Elemente.

MapObj := Map(Schlüssel1, Wert1, Schlüssel2, Wert2, ...)

Dies ist das gleiche wie, als würde man jedes Element mit MapObj[Schlüssel] := Wert setzen, außer dass __Item nicht aufgerufen wird und Capacity automatisch angepasst wird, um zu vermeiden, dass es während eines einzigen Aufrufs mehrfach erweitert wird.

Parameter werden durch __New definiert.

Methoden

Clear

Entfernt alle Schlüssel-Wert-Paare aus einem Map-Objekt.

MapObj.Clear()

Clone

Gibt eine flache Kopie eines Map-Objekts zurück.

Klon := MapObj.Clone()

Alle Schlüssel-Wert-Paare werden in das neue Map-Objekt kopiert. Objektreferenzen werden kopiert (wie bei einer normalen Zuweisung), nicht die Objekte selbst.

Eigene Eigenschaften, eigene Methoden und Basis werden wie bei Obj.Clone kopiert.

Delete

Entfernt ein Schlüssel-Wert-Paar aus einem Map-Objekt.

MapObj.Delete(Schlüssel)
Schlüssel

Typ: Integer, Objekt oder Zeichenkette

Beliebiger einzelner Schlüssel. Wenn das Map-Objekt diesen Schlüssel nicht enthält, wird ein UnsetItemError ausgelöst.

Gibt den entfernten Wert zurück.

Get

Gibt den Wert eines zugehörigen Schlüssels oder einen Standardwert zurück.

Wert := MapObj.Get(Schlüssel , Standardwert)

Diese Methode bewirkt Folgendes:

  • Gib den Wert zurück, der mit Schlüssel verknüpft ist, falls gefunden.
  • Gib den Wert des Standardwert-Parameters zurück, falls angegeben.
  • Gib den Wert von MapObj.Default zurück, falls definiert.
  • Löse einen UnsetItemError aus.

Wenn Standardwert weggelassen wird, ist dies äquivalent zu MapObj[Schlüssel], außer dass __Item nicht aufgerufen wird.

Has

Gibt True zurück, wenn der angegebene Schlüssel einen zugeordneten Wert innerhalb eines Map-Objekts hat, ansonsten False.

MapObj.Has(Schlüssel)

Set

Setzt null oder mehr Elemente.

MapObj.Set(Schlüssel, Wert, Schlüssel2, Wert2, ...)

Dies ist das gleiche wie, als würde man jedes Element mit MapObj[Schlüssel] := Wert setzen, außer dass __Item nicht aufgerufen wird und Capacity automatisch angepasst wird, um zu vermeiden, dass es während eines einzigen Aufrufs mehrfach erweitert wird.

Gibt das Map-Objekt zurück.

__Enum

Enumeriert Schlüssel-Wert-Paare.

For Schlüssel , Wert in MapObj

Gibt einen neuen Enumerator zurück. Diese Methode wird für gewöhnlich nicht direkt aufgerufen. Stattdessen wird das Map-Objekt direkt an eine For-Schleife übergeben, die __Enum einmal aufruft und dann den Enumerator einmal für jede Wiederholung der Schleife aufruft. Jeder Aufruf des Enumerators gibt den nächsten Schlüssel und/oder Wert zurück. Die Variablen der For-Schleife entsprechen den folgenden Parametern des Enumerators:

Schlüssel

Typ: Integer, Objekt oder Zeichenkette

Der Schlüssel.

Wert

Der Wert.

__New

Setzt Elemente. Äquivalent zu Set.

MapObj.__New(Schlüssel, Wert, Schlüssel2, Wert2, ...)

Diese Methode existiert zur Unterstützung von Call und ist nicht für den direkten Aufruf vorgesehen. Siehe Konstruktion und Destruktion.

Eigenschaften

Count

Ruft die Anzahl von Schlüssel-Wert-Paaren in einem Map-Objekt ab.

Anzahl := MapObj.Count

Capacity

Ermittelt oder setzt die aktuelle Kapazität eines Map-Objekts.

MapObj.Capacity := MaxElemente
MaxElemente := MapObj.Capacity
MaxElemente

Typ: Integer

Die maximale Anzahl von Schlüssel-Wert-Paaren, die das Map-Objekt beinhalten soll, bevor es automatisch erweitert werden muss. Wenn ein Wert kleiner als die aktuelle Anzahl von Schlüssel-Wert-Paaren gesetzt wird, wird stattdessen dieser Wert verwendet und ungenutzter Speicher freigegeben.

CaseSense

Ermittelt oder setzt die Groß-/Kleinschreibung-Sensitivität eines Map-Objekts.

MapObj.CaseSense := Einstellung
Einstellung := MapObj.CaseSense
Einstellung

Typ: Zeichenkette

Geben Sie einen der folgenden Werte an:

"On": Die Suche von Schlüsseln erfolgt Groß-/Kleinschreibung-sensitiv. Standardeinstellung.

"Off": Die Großbuchstaben A bis Z und die entsprechenden Kleinbuchstaben werden gleichwertig behandelt.

"Locale": Die Suche von Schlüsseln erfolgt nicht Groß-/Kleinschreibung-sensitiv, gemäß den Regeln der aktuellen Sprach- und Regionseinstellungen des Benutzers. In den meisten englischen und westeuropäischen Regionen beispielsweise werden die Großbuchstaben A bis Z, einschließlich ASCII-fremde Zeichen wie Ä und Ü, und die entsprechenden Kleinbuchstaben gleichwertig behandelt. Locale ist 1- bis 8-mal langsamer als Off, abhängig von den zu vergleichenden Zeichenketten.

Beim Zuweisen eines Wertes können auch die Zeichenketten "1" und "0" verwendet werden.

Wenn das Map-Objekt nicht leer ist und dieser Eigenschaft etwas zugewiesen wird, wird eine Ausnahme ausgelöst.

Default

Definiert den Standardwert, der zurückgegeben werden soll, wenn ein Schlüssel nicht gefunden wird.

MapObj.Default := Wert

Diese Eigenschaft ist standardmäßig nicht vorhanden, kann aber via Skript definiert werden. Wenn das Skript einen Schlüssel abfragt, der nicht gefunden wird, prüft das Map-Objekt, ob diese Eigenschaft vorhanden ist, bevor sie einen UnsetItemError auslöst. Die Eigenschaft kann mit den üblichen Mitteln implementiert werden, einschließlich einer dynamischen Eigenschaft oder einer Metafunktion, aber um festzustellen, welcher Schlüssel abgefragt wurde, müsste stattdessen __Item oder Get überschrieben werden.

__Item

Ermittelt oder setzt den Wert eines Schlüssel-Wert-Paares.

Wert := MapObj[Schlüssel]
MapObj[Schlüssel] := Wert
Schlüssel

Typ: Integer, Objekt oder Zeichenkette

Beim Abrufen eines Wertes muss Schlüssel ein einzigartiger Wert sein, der zuvor einem anderen Wert zugeordnet wurde. Es wird ein UnsetItemError ausgelöst, wenn Schlüssel keinen zugeordneten Wert innerhalb des Map-Objekts hat, es sei denn, es ist eine Standardwert-Eigenschaft definiert, dann wird deren Wert zurückgegeben.

Beim Zuweisen eines Wertes kann Schlüssel ein beliebiger Wert sein, der mit Wert assoziiert werden kann; das heißt, dass der Schlüssel genutzt wird, um später auf Wert zugreifen zu können. Float-Schlüssel werden automatisch in eine Zeichenkette umgewandelt.

Der Eigenschaftsname __Item wird typischerweise weggelassen, wie oben gezeigt, ist aber nützlich, um die Eigenschaft zu überschreiben.
