ToolTip

Erstellt ein Fenster irgendwo auf dem Bildschirm, das immer im Vordergrund ist.

ToolTip Text, X, Y, WelcherToolTip

Parameter

Text

Typ: Zeichenkette

Wenn dieser Parameter weggelassen wird oder leer ist, wird der Tooltip (sofern vorhanden) versteckt. Ansonsten kann eine Zeichenkette angegeben werden, die im Tooltip angezeigt werden soll. Um einen Tooltip mehrzeilig zu machen, trennen Sie jede Zeile mit einem Zeilenumbruchszeichen (`n), z. B. Zeile1`nZeile2.

Wenn der Text relativ lang ist, kann er via Fortsetzungsbereich in mehreren kurzen Zeilen aufgeteilt werden, um die Übersichtlichkeit und Wartbarkeit des Skripts zu erhöhen.

X, Y

Typ: Integer

Die X- und Y-Koordinate der oberen linken Ecke des Tooltips, relativ zum aktiven Fenster (mit CoordMode "ToolTip" können die Koordinaten relativ zum Bildschirm gemacht werden). Lässt man die Koordinaten weg, wird der Tooltip in der Nähe des Mauszeigers angezeigt.

WelcherToolTip

Typ: Integer

Dieser Parameter dient dazu, mehrere Tooltips gleichzeitig anzeigen zu lassen. Eine Nummer zwischen 1 und 20 kennzeichnet, auf welchem Tooltip-Fenster die Operation durchgeführt werden soll. Wenn keine Nummer angegeben ist, wird standardmäßig 1 verwendet (das erste Fenster).

Rückgabewert

Typ: Integer oder leere Zeichenkette

Wenn ein Tooltip angezeigt oder aktualisiert wird, gibt diese Funktion die HWND-Nummer (eindeutige ID) des Tooltip-Fensters zurück. Diese Nummer kann genutzt werden, um den Tooltip zu verschieben oder Tooltip-Steuerelementmeldungen zu senden.

Wenn Text weggelassen wird oder leer ist, wird eine leere Zeichenkette zurückgegeben.

Bemerkungen

Ein typischer Tooltip sieht wie folgt aus:

Wenn die angegebene X- und Y-Koordinate dazu führen, dass der Tooltip außerhalb des Bildschirms oder unter Windows 8 oder höher außerhalb des Arbeitsbereichs des Anzeigegeräts liegt, wird der Tooltip neu positioniert, um vollständig sichtbar zu bleiben.

Der Tooltip wird solange angezeigt, bis eine der folgenden Bedingungen erfüllt ist:

  • Das Skript wird terminiert.
  • Die ToolTip-Funktion wird mit leerem Text-Parameter erneut ausgeführt.
  • Der Benutzer hat den Tooltip angeklickt (dieses Verhalten variiert je nach Betriebssystem).

Mithilfe der OwnDialogs-Option kann ein Tooltip einem GUI-Fenster untergeordnet werden. Tooltips dieser Art werden automatisch zerstört, sobald ihr übergeordnetes Fenster zerstört wird.

Beispiele

Zeigt einen mehrzeiligen Tooltip an einer bestimmten Position im aktiven Fenster an.

ToolTip "Mehrzeiliger`nTooltip", 100, 150

Versteckt einen Tooltip nach einer gewissen Zeit, ohne Sleep verwenden zu müssen (das den aktuellen Thread anhalten würde).

ToolTip "Zeitgesteuerter Tooltip`nEr wird 5 Sekunden lang angezeigt."
SetTimer () => ToolTip(), -5000
