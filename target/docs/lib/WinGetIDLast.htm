WinGetIDLast

Gibt die eindeutige ID-Nummer des letzten/untersten Fensters zurück, wenn mehr als ein Fenster gefunden wird.

LetzteID := WinGetIDLast(FensterTitel, FensterText, AusnahmeTitel, AusnahmeText)

Parameter

FensterTitel, FensterText, AusnahmeTitel, AusnahmeText

Typ: Zeichenkette, Integer oder Objekt

Wenn jeder dieser Parameter leer ist oder weggelassen wird, wird das Zuletzt Gefundene Fenster verwendet. Andernfalls geben Sie für FensterTitel einen Fenstertitel oder andere Kriterien zur Identifizierung des Zielfensters und/oder für FensterText eine Teilzeichenkette aus einem Textelement des Zielfensters an (Textelemente können mit dem internen Tool "Window Spy" aufgedeckt werden). Versteckte Textelemente werden erkannt, wenn DetectHiddenText eingeschaltet wurde. Fenster, deren Titel oder Texte AusnahmeTitel bzw. AusnahmeText enthalten, werden ignoriert.

Rückgabewert

Typ: Integer

Diese Funktion gibt die HWND-Nummer (eindeutige ID) des letzten/untersten Fensters zurück, wenn mehr als ein Fenster gefunden wird.

Fehlerbehandlung

Es wird ein TargetError ausgelöst, wenn das Fenster nicht gefunden werden konnte.

Bemerkungen

Wenn nur ein Fenster gefunden wird, verhält sich diese Funktion wie WinGetID.

Die Gültigkeit einer ID-Nummer ist an die Lebensdauer des zugehörigen Fensters gebunden. Mit anderen Worten: Wenn eine Anwendung neu gestartet wird, erhalten all ihre Fenster neue ID-Nummern.

Mit MouseGetPos kann die ID des Fensters unter dem Mauszeiger abgerufen werden. Um die HWND-Nummer eines Steuerelements abzurufen (z.B. für PostMessage, SendMessage oder DllCall), können Sie ControlGetHwnd oder MouseGetPos nutzen.

Fenstertitel und -texte sind Groß-/Kleinschreibung-sensitiv. Versteckte Fenster werden nur erkannt, wenn DetectHiddenWindows eingeschaltet ist.
