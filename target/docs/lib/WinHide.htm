WinHide

Versteckt ein bestimmtes Fenster.

WinHide FensterTitel, FensterText, IgnoriereTitel, IgnoriereText

Parameter

FensterTitel

Typ: Zeichenkette, Integer oder Objekt

Der Titel oder andere Möglichkeiten zum Identifizieren des Zielfensters. Weitere Informationen finden Sie unter FensterTitel-Parameter.

FensterText

Typ: Zeichenkette

Eine Zeichenkette, die in einem der Textelemente des Zielfensters enthalten ist (eventuell vorhandene Textelemente können mit dem internen Tool Window Spy aufgedeckt werden). Versteckte Textelemente werden nur erkannt, wenn DetectHiddenText eingeschaltet ist.

IgnoriereTitel

Typ: Zeichenkette

Alle Fenster, die diesen Wert in ihren Titeln enthalten, werden ignoriert.

IgnoriereText

Typ: Zeichenkette

Alle Fenster, die diesen Wert in ihren Textelementen enthalten, werden ignoriert.

Fehlerbehandlung

Es wird ein TargetError ausgelöst, wenn das Fenster nicht gefunden werden konnte, es sei denn, der Gruppenmodus wird verwendet.

Bemerkungen

Mit WinShow kann ein verstecktes Fenster wieder sichtbar gemacht werden (DetectHiddenWindows kann entweder On oder Off sein, um das zu ermöglichen).

Diese Funktion agiert nur mit dem obersten übereinstimmenden Fenster, es sei denn, FensterTitel ist ahk_group GruppeName, dann sind alle Fenster in der Gruppe betroffen.

Die Explorer-Taskleiste kann wie folgt versteckt / sichtbar gemacht werden:

WinHide "ahk_class Shell_TrayWnd"
WinShow "ahk_class Shell_TrayWnd"

Siehe auch

WinShow, SetTitleMatchMode, DetectHiddenWindows, Zuletzt Gefundenes Fenster, Win-Funktionen

Beispiele

Öffnet den Texteditor, wartet, bis er existiert, versteckt ihn für kurze Zeit und macht ihn wieder sichtbar.

Run "notepad.exe"
WinWait "Unbenannt - Editor"
Sleep 500
WinHide ; Verwendet das von WinWait gefundene Fenster.
Sleep 1000
WinShow ; Verwendet das von WinWait gefundene Fenster.
