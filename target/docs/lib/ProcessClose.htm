ProcessClose

Erzwingt die Schließung des ersten gefundenen Prozesses.

ProcessClose PIDOderName

Parameter

PIDOderName

Typ: Integer oder Zeichenkette

Geben Sie entweder eine Nummer (die PID) oder einen Prozessnamen an:

PID (Prozess-ID): Eine Nummer zur eindeutigen Identifizierung eines Prozesses (die Gültigkeit dieser Nummer ist an die Lebensdauer des Prozesses gebunden). Mit der Run-Funktion kann die PID eines neu gestarteten Prozesses ermittelt werden. Im Vergleich dazu kann mit WinGetPID die PID eines Fensters ermittelt werden. Mit ProcessExist kann auch eine PID abgerufen werden.

Name: Der Name eines Prozesses (ohne Pfad), den üblicherweise auch seine ausführbare Datei hat, z.B. notepad.exe oder winword.exe. Da ein Name mit mehreren laufenden Prozessen übereinstimmen kann, wird nur der erste Prozess berücksichtigt. Der Name ist nicht Groß-/Kleinschreibung-sensitiv.

Rückgabewert

Typ: Integer

Diese Funktion gibt die Prozess-ID (PID) des angegebenen Prozesses zurück. Wenn ein passender Prozess nicht gefunden wird oder nicht manipuliert werden kann, wird 0 zurückgegeben.

Bemerkungen

Da der Prozess abrupt terminiert wird - mit der Chance, dass seine Ausführung an einem kritischen Punkt unterbrochen wird oder dass ungespeicherte Daten innerhalb seiner Fenster (sofern er welche hat) verloren gehen - sollte diese Funktion nur verwendet werden, wenn keines der Fenster des Prozesses via WinClose geschlossen werden kann.

Run, WinClose, WinKill, Process-Funktionen, Win-Funktionen

Beispiele

Erzwingt die Schließung von Notepad (seien Sie gewarnt, dass jegliche ungespeicherte Daten verloren gehen).

ProcessClose "notepad.exe"
