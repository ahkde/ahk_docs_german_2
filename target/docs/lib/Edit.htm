Edit

Die Edit-Funktion öffnet das aktuelle Skript mithilfe des zugeordneten Verbs "edit", das in der Registry angegeben ist (oder mit dem Windows-Texteditor, wenn kein Verb vorhanden ist). Stellt sich heraus, dass ein Editorfenster das Skript bereits offen hat (abhängig vom Fenstertitel), wird dieses Fenster aktiviert, anstatt eine neue Instanz des Editors zu öffnen.

Programme, Skripte oder Befehlszeilen, die standardmäßig mit dem Verb "edit" ausgeführt werden, können über die Editor settings im Dash geändert werden.

Diese Funktion hat keine Wirkung, wenn sie innerhalb eines kompilierten Skripts aufgerufen wird.

Verschiedene Texteditoren unterstützen eine Syntaxhervorhebung für AutoHotkey (siehe unten). Des Weiteren kann eine kontextabhängige Hilfe für AutoHotkey-Funktionen mit diesem Beispiel aktiviert werden. Um die eigene Produktivität zu erhöhen, kann ein Auto-Vervollständigungs-Tool wie das Skript von boiler oder das Skript von Helgef genutzt werden, die bei fast jedem Editor funktionieren. Es überwacht, was Sie eingeben, und zeigt Menüs und Parameterlisten an, die Ihnen einen Teil der Eingabe abnimmt und Sie an die Reihenfolge der Parameter erinnert.

Wenn die Befehlszeilenverwendung Ihres Editors ungefähr wie Editor.exe "Kompletter Pfad des Skripts.ahk" aussieht, können Sie ihn wie folgt als Standardeditor für AHK-Dateien festlegen. Wenn Sie das Skript ausführen, werden Sie aufgefordert, die EXE-Datei Ihres Editors auszuwählen.

Editor := FileSelect(2,, "Wählen Sie Ihren Editor aus", "Programme (*.exe)")
if Editor = ""
    ExitApp
RegWrite "REG_SZ", "HKCR\AutoHotkeyScript\Shell\Edit\Command",, Format('"{1}" "%1"', Editor)

Editoren mit AutoHotkey-Unterstützung

Jeder Texteditor kann zum Editieren eines AutoHotkey-Skripts verwendet werden, aber Editoren, die AutoHotkey direkt unterstützen (oder so konfiguriert werden können), können das Lesen, Editieren und Testen von Skripten wesentlich erleichtern. AutoHotkey-unterstützende Editoren können folgendes bereitstellen:

  • Syntaxhervorhebung, wie sie in dieser Dokumentation verwendet wird. Bei der Syntaxhervorhebung werden Wörter, Symbole und Codesegmente farblich gekennzeichnet, um ihre Bedeutung zu verdeutlichen. Zum Beispiel werden direkt geschriebene Texte, Kommentare und Variablennamen in verschiedenen Farben angezeigt.
  • Autovervollständigung, die in der Regel eine Liste von Vorschlägen anzeigt, wenn Sie beginnen, den Namen einer bekannten Funktion oder Variable einzugeben.
  • Calltips können Ihnen die Parameter einer Funktion anzeigen, während Sie den Code zum Aufrufen der Funktion schreiben.
  • Interaktives Debuggen, z. B. um das Skript Zeile für Zeile durchzugehen und die Variablen bei jedem Schritt zu untersuchen, oder um Variablen oder Objekte einzusehen und zu ändern, jenseits dessen, was ListVars ermöglicht.

Empfehlungen:

  • SciTE4AutoHotkey ist einfach zu installieren, verhältnismäßig schlank und unterstützt AutoHotkey v1 und v2 ohne weitere Konfiguration.
  • VS Code (plus Erweiterungen) bietet ein noch höheres Maß an Unterstützung und ein breiteres Spektrum an Features, kann aber sehr ressourcenintensiv sein.

SciTE4AutoHotkey ist eine benutzerdefinierte Version des Texteditors SciTE. Seine Features umfassen:

  • Syntaxhervorhebung
  • Autovervollständigung
  • Calltips
  • Intelligente automatische Einrückung
  • Codefaltung
  • Interaktives Debuggen
  • Skript durch Drücken eines Hotkeys ausführen
  • Weitere Tools für AutoHotkey-Skripte

SciTE4AutoHotkey kann hier gefunden werden: https://www.autohotkey.com/scite4ahk/

Visual Studio Code (VS Code) kann durch die Installation von Erweiterungen mit einem hohen Maß an Unterstützung für AutoHotkey konfiguriert werden.

AutoHotkey2 Language Support bietet zahlreiche Features, darunter:

  • Syntaxhervorhebung
  • Autovervollständigung
  • Calltips
  • Intelligente automatische Einrückung
  • Codefaltung
  • Skript durch Drücken eines Hotkeys ausführen
  • Echtzeit-Diagnose (Erkennung häufiger Fehler)
  • Code formatieren/aufräumen

Weitere Hinweise:

  • Diese Erweiterung unterstützt nur AutoHotkey v2, kann aber auch v1-Skripte erkennen und automatisch auf eine v1-Erweiterung umschalten, wenn so eine installiert ist.
  • Diese Erweiterung kann auch mit anderen Editoren verwendet werden, wie z. B. vim, neovim und Sublime Text 4. Einzelheiten finden Sie unter Use in other editors. VS Code bietet jedoch wahrscheinlich die beste Erfahrung und die einfachste Einrichtung.

vscode-autohotkey-debug bietet Unterstützung für interaktives Debuggen von v1- und v2-Skripten.

Notepad++ kann so konfiguriert werden, dass es die folgenden Features unterstützt:

Instruktionen finden Sie unter Setup Notepad++ for AutoHotkey.
