Wie man Beispielcodes ausführt

Um schnell mit AutoHotkey loszulegen, ist es am einfachsten, einen Beispielcode zu kopieren, ihn auszuprobieren und nach Bedarf anzupassen.

Innerhalb dieser Dokumentation finden Sie viele Beispiele in Codeblöcken. Zum Beispiel:

MsgBox "Hallo, Welt!"

Die meisten (aber nicht alle) Beispiele können direkt ausgeführt werden, um ihre Wirkung sofort zu sehen. Es gibt grundsätzlich zwei Wege, um das zu erreichen:

  1. Laden Sie den Code als Datei herunter. Sofern es Ihr Browser unterstützt, können Sie jeden Codeblock (wie der oben) als Skriptdatei herunterladen, indem Sie den Mauszeiger über den Codeblock bewegen und oben rechts auf die Schaltfläche ↓ klicken.
  2. Kopieren Sie den Code in eine Datei. In der Regel ist es am besten, eine neue Datei zu erstellen, damit der vorhandene Code nicht mit dem Beispielcode in Konflikt gerät. Sobald die Datei erstellt ist, öffnen Sie sie zur Bearbeitung und fügen Sie den Code ein.

Die Datei ausführen: Sobald Sie den Code in eine Skriptdatei (.ahk) kopiert haben, genügt normalerweise ein Doppelklick auf die Datei, um den Code auszuführen; aber es gibt auch andere Methoden.

Hotkeys zuweisen

Manchmal ist es zum Testen von Code einfacher, wenn Sie ihn zuerst einem Hotkey zuweisen. Schauen Sie sich zum Beispiel den folgenden Code zum Maximieren des aktiven Fensters an:

WinMaximize "A"

Wenn Sie diesen Code in eine Datei kopieren und die Datei mit einem Doppelklick ausführen, wird wahrscheinlich das Datei-Explorer-Fenster, das die Datei enthält, maximiert. Alternativ können Sie den Code einem Hotkey zuweisen, um seine Wirkung auf ein beliebiges Fenster zu testen. Zum Beispiel:

^1::WinMaximize "A"

Jetzt können Sie Ihr Testfenster aktivieren und mit Strg+1 maximieren.

Weitere Informationen zu Hotkeys finden Sie unter Wie man Hotkeys schreibt.

Sicherheitsmaßnahmen

Wenn Sie einen Fehler in einem Skript machen, kann es passieren, dass der Computer schwieriger zu bedienen ist. Zum Beispiel wird der Hotkey n:: jedes Mal ausgelöst, wenn Sie N drücken, und verhindert, dass Sie dieses Zeichen eingeben können. Um das rückgängig zu machen, müssen Sie nur das Skript beenden. Rechtsklicken Sie hierfür auf das Tray-Symbol des Skripts und wählen Sie Exit aus.

Tasten können "klemmen", wenn Sie ein Taste-Unten-Ereignis, aber kein Taste-Oben-Ereignis senden. In diesem Fall hilft es nicht unbedingt, das Skript zu beenden, da das Betriebssystem immer noch glaubt, dass die Taste gedrückt gehalten wird. Grundsätzlich können Sie die Taste durch Drücken der Taste "lockern".

Wenn ein Skript in eine Endlosschleife gerät oder aus anderen Gründen schwer zu stoppen ist, können Sie sich abmelden oder den Computer notfalls herunterfahren. Wenn Sie sich abmelden, werden alle laufenden Programme aus Ihrer Sitzung beendet, einschließlich AutoHotkey. In einigen Fällen müssen Sie möglicherweise auf "Trotzdem abmelden" oder "Trotzdem herunterfahren" klicken, wenn ein Skript oder Programm das Herunterfahren verhindert.

Neu laden

Nachdem Sie das Skript gestartet haben, werden Änderungen an der Skriptdatei nicht automatisch wirksam. Damit diese wirksam werden, müssen Sie das Skript neu laden. Verwenden Sie hierfür das Tray-Symbol des Skripts oder die Reload-Funktion, die Sie über einen Hotkey aufrufen können. In vielen Fällen ist es auch möglich, das Skript einfach erneut auszuführen, aber das hängt von der #SingleInstance-Einstellung des Skripts ab.

Die richtigen Tools

Programmieren lernen ist oft ein sich wiederholender Prozess: Man nimmt etwas Code, macht eine kleine Änderung, testet den Code und wiederholt das Ganze. Dieser Prozess ist schneller und produktiver, wenn Sie einen Texteditor mit AutoHotkey-Unterstützung verwenden. Die Unterstützung variiert je nach Editor, aber die wichtigsten Funktionen sind (meiner Meinung nach):

  • Die Möglichkeit, das Skript mit einem Tastaturkurzbefehl (z.B. F5) auszuführen.
  • Syntaxhervorhebung, damit der Code leichter zu lesen (und zu schreiben) ist.

Empfehlungen finden Sie unter Editoren mit AutoHotkey-Unterstützung oder Editors-Unterforum.
